## 📊 ОБЩАЯ ИНФОРМАЦИЯ

**Проект:** ArtSoul Marketplace - децентрализованный маркетплейс для цифрового искусства  
**Технологии:**
- Frontend: Vanilla JS + React JSX (для gallery.html, profile.html)
- Backend: Supabase (PostgreSQL + Storage + Auth)
- Blockchain: Ethereum (Base Sepolia + Ethereum Sepolia)
- Web3: Reown AppKit (WagmiAdapter)
- Хранилище: Supabase Storage (mock IPFS)

**Статистика кода:**
- ~9,227 строк кода (JS + HTML)
- 5 HTML страниц
- 10+ JavaScript модулей
- 4 основные таблицы в БД

# 3. Добавить UNIQUE constraint на votes
# Скопировать содержимое: supabase-migration-votes-unique.sql

**Почему важно:**
- Без blockchain_id невозможна синхронизация с блокчейном
- Без RLS политик любой может изменить любой профиль
- Без UNIQUE constraint возможно дублирование голосов

- Проблема: Подпись не проверяется на бэкенде

- Решение: Создать Edge Function для проверки подписи

**Отсутствие rate limiting:**
- Проблема: Нет ограничений на частоту вызовов

- Решение: Добавить rate limiting через Supabase или Cloudflare

- Проблема: Каждый раз загружаются все данные
- Решение: Добавить кэширование через localStorage или React Query

- Решение: Добавить спиннеры или изменение текста кнопки

**Кнопки без disabled состояния:**

- Решение: Добавить disabled состояние во время async операций

## 🎯 ЛОГИКА РАБОТЫ МАРКЕТПЛЕЙСА

### КОНЦЕПЦИЯ ПРОЕКТА:

**ArtSoul Marketplace** - это децентрализованный маркетплейс для цифрового искусства, где:
- Художники загружают свои работы как NFT
- Работы продаются через аукционы
- Сообщество голосует за лучшие работы
- Все транзакции на блокчейне (Base Sepolia / Ethereum Sepolia)

### ПРИНЦИП РАБОТЫ:

**1. Загрузка артворка (Upload Flow):**
```
Художник → Выбирает файл → Заполняет форму → Загружает в Storage
→ Генерируется IPFS hash → Создается metadata → Загружается на блокчейн
→ Получаем blockchain_id → Сохраняем в БД → Статус: DRAFT
```

**2. Создание аукциона (Auction Flow):**
```
Художник → Открывает профиль → Выбирает DRAFT артворк → Создает аукцион
→ Проверяется статус на блокчейне → Создается аукцион на блокчейне
→ Обновляется статус в БД: AUCTION → Триггер устанавливает end_time (3 дня)
```

**3. Размещение ставки (Bidding Flow):**
```
Покупатель → Открывает артворк → Вводит сумму → Размещает ставку
→ Проверяется минимум (startingPrice или highestBid * 1.05)
→ Отправляется транзакция на блокчейн → Сохраняется в БД
→ Триггер обновляет winner_id и winning_bid_amount
```

**4. Завершение аукциона (End Auction Flow):**
```
Художник/Покупатель → Нажимает "End Auction" → Проверяется время
→ Вызывается endAuction() на блокчейне → NFT передается победителю
→ Обновляется owner_address в БД → Статус: SOLD
→ Артворк появляется в "Collected" у победителя
```

**5. Голосование (Voting Flow):**
```
Пользователь → Открывает артворк → Нажимает "Vote"
→ Проверяется подключение кошелька → Проверяется дубликат
→ Валидируется UUID и адрес → Вставляется запись в votes
→ Обновляется счетчик голосов → Артворк поднимается в рейтинге
```

**6. Профиль (Profile Flow):**
```
Пользователь → Подключает кошелек → Автоматически создается профиль
→ Username: User{first6chars} → Avatar: DiceBear API
→ Профиль синхронизируется между устройствами (wallet_address как ключ)
→ Можно редактировать: username, bio, avatar, social links
```

### РОЛИ ПОЛЬЗОВАТЕЛЕЙ:

**1. Художник (Creator):**
- Загружает артворки
- Создает аукционы
- Завершает аукционы
- Получает оплату за проданные работы
- Может удалить DRAFT артворки
- Может перевыставить неудачные аукционы

**2. Покупатель (Collector):**
- Просматривает галерею
- Голосует за артворки
- Размещает ставки на аукционах
- Получает NFT при победе
- Видит свои покупки в "Collected"

**3. Посетитель (Visitor):**
- Просматривает галерею
- Видит детали артворков
- Не может голосовать или покупать
- Должен подключить кошелек для действий

### ЭКОНОМИЧЕСКАЯ МОДЕЛЬ:

**Комиссии:**
- Platform fee: 2.5% (250 basis points)
- Вычитается из финальной цены аукциона
- Накапливается в контракте
- Владелец контракта может вывести через withdrawFees()

**Распределение средств при продаже:**
```
Финальная ставка: 1.0 ETH
Platform fee (2.5%): 0.025 ETH → Контракт
Художник получает: 0.975 ETH
```

**Creator Value vs Community Value:**
- Creator Value: Начальная цена, установленная художником
- Community Value: Определяется голосованием (пока не используется)
- System Value: Накопленные комиссии платформы
