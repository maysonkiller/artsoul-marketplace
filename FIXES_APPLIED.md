# Исправления ArtSoul - 28 апреля 2026

## Проблемы которые были исправлены:

### 1. ✅ Постоянные запросы подписи при навигации
**Проблема:** При каждом переходе между страницами кошелек открывал окно с запросом подписи транзакции.

**Решение:** 
- Убрана автоматическая аутентификация при подключении кошелька в `appkit-init.js:336-352`
- Теперь аутентификация происходит только когда пользователь пытается сохранить профиль или загрузить artwork
- Добавлена проверка существующей сессии в `supabase-auth.js:36-44` перед запросом новой подписи

### 2. ✅ Ошибка RLS при сохранении профиля
**Проблема:** `Error saving profile: new row violates row-level security policy for table "profiles"`

**Решение:**
- Создан новый SQL скрипт `supabase-rls-fix.sql` с упрощенными RLS политиками
- Политики теперь проверяют только `auth.uid() IS NOT NULL` вместо сложных JWT claims
- Проверка владения wallet_address происходит на уровне приложения

**ВАЖНО:** Нужно выполнить SQL скрипт в Supabase SQL Editor:
1. Открыть https://supabase.com/dashboard/project/bexigvqrunomwtjsxlej/sql
2. Скопировать содержимое файла `supabase-rls-fix.sql`
3. Выполнить скрипт

### 3. ✅ Дисконнект при переходе на profile.html
**Проблема:** При переходе на страницу профиля кошелек отключался и просил подключиться заново.

**Решение:**
- Убрана автоматическая аутентификация, которая вызывала конфликт
- Теперь кошелек остается подключенным при навигации между страницами
- Аутентификация вызывается только через `window.ensureAuthenticated()` когда нужно

## Как теперь работает аутентификация:

1. **Подключение кошелька** - пользователь подключает кошелек через AppKit (без подписи)
2. **Навигация** - можно свободно переходить между страницами, кошелек остается подключенным
3. **Сохранение данных** - при попытке сохранить профиль или загрузить artwork вызывается `ensureAuthenticated()`, который:
   - Проверяет существующую сессию
   - Если сессии нет - запрашивает подпись один раз
   - Сохраняет сессию в localStorage
4. **Повторные действия** - подпись больше не запрашивается, используется сохраненная сессия
